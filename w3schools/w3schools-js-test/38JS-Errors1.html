JavaScript Errors - Throw and Try to Catch

The try statement lets you test a block of code for errors. In other words, the try statement allows you to define a block of code to be tested for errors while it is being executed.

The catch statement lets you handle the error. In other words, the catch statement allows you to define a block of code to be executed, if an error occurs in the try block.

The throw statement lets you create custom errors.

The finally statement lets you execute code, after try and catch, regardless of the result.

Errors Will Happen!

When executing JavaScript code, different errors can occur. Errors can be coding errors made by the programmer, errors due to wrong input, and other unforeseeable things.

JavaScript Throws Errors

When an error occurs, JavaScript will normally stop and generate an error message. The technical term for this is: JavaScript will throw an exception (throw an error).

JavaScript will actually create an Error object with two properties: name and message.
